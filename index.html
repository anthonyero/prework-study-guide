HTML

  • The head section of a document contains important metadata including the title of the page that will appear as text displayed on the tab.
  • The body element defines the body of the document which contains the content that is visible to the user. All of the content that we want to display to users must be inside the body element.
  • Although there are some elements that are found within a set of opening and closing tags such as head, body, and p, others such as img, br contain all relevent text within one set of "< >"
  • Notice that there are child elements and parent elements
  • The biggest heading h1 should only appear once on each page: we should only have one h1 per page

Git

  • git status: checks what branch we are currently on
  • git checkout: switches to a new branch. This reminds me of the "cd" functionality in the terminal
  • git checkout -b branch-name: creates a new branch and switches to it. The "-b" tells Git to create a new branch
  • When creating a new branch, it is good practice to give the branch a specific but concise name and preface it with whether the branch is for a feature. For example, the creation and movement to a new branch may look like "git checkout -b feature/adding-new-branch"
  • git pull origin main: Pulls down the latest main branch. To my knowledge, we want to ensure we have the latest repo/branch before we start working to ensure we do not advance on outdated code
  • The Git Flow
    1. Make sure that we are on the correct branch from the command line. Use the git status and git checkout branch-name to get there
    2. We have to add the changed files in our working directory to the staging area of Git. Use the following command in the terminal to stage changes: git add -A. We use "add -A" command to commmit all files in the working branch
    3. Now that all of our changes have been staged, we can commit them to our repository. Use the following command in the terminal to commit the changes: git commit -m "comment about updates in these changes". We use the "-m" flag to associate a message with our commit. Commit messages should be short descriptive messages of the changes implemented. Once completed, our Git repository is now up to date with our working directory. We should receive output in the terminal suggesting changes/insertions and, I'm guessing, removals when appropriate
    4. Next, make sure that our current branch is up to date before opening a pull request. We need to sync our local branch with the remote branch in Github. This is especially important in a team environment in case a team member has added a new feature to the project. The process of syncing and emrging code into the Github repo is how the project gets continually updated with more features and content from different team members. Use the following command in the terminal to pull in the base branch: git pull origin main. It's a good habit to always pull the base branch into your feature branch before pushing up any code. You should see something simliar to the following in your terminal if your branches are in sync: "Already up to date."
    5. Push all of our changes by using the following command: git push origin branch-name
    6. Now, navigate to the repository on Github in the browser. We need merge our feature branch into the base branch so that in a real work situation other team members can upate their feature branches with an updated version of the project. Once you are in the repository, navigate to the relevant branch using the navigation tabs
    7. Click the "Pull requests" tab at the top
    8. Once on the pull requests page, find and click the "New pull request" button
    9. Now we need to compare the two branches that we want to merge. The base branch that we want to merge into should come first. In our case, this is the "main" branch. We want to merge our "feature/branch-name", so that is our compare branch.
    10. Once on the pull request form, give your PR a title and a short description of the changes
    11. Click on the "Create pull request button"
    12. It is time to merge our new feature into our main codebase! Click the "Merge pull request" button and confirm
    13. Navigate back to the "main" branch in GitHub and you should see that the relevant files/changes in the main branch
    14. Let's open our terminal and update our local repository with the new merged-in feature. Use the following commands in the terminal git checkout main then git pull origin main
    15. Close the relevant issue on GitHub since we have merged in the feature associated with it.
